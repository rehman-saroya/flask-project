Software Development Life Cycle (SDLC)

  • Planning: Clarify what needs to be built.
  • Design: Engineer the system structure.
  • Implementation: Develop the code.
  • Testing: Confirm the code works.
  • Deployment: Deliver the software.
  • Maintenance: Improve and fix issues over time.

The software development lifecycle is important as it is needed to build a software application. This development goes through a procedure in which there are several stages. Additionally, developers add features that are needed while also fixing bugs in the software. The first stage includes the planning phase. The planning phase includes duties like arranging, examining and granting. In this stage. The team of developers recieves information from customers,manangers and experts to create a document that is needed for software demands. This document has expectations that needs to be met through commmon goals that help when the project is planned. The second stage includes the design phase. In the design phase, software engineers try to find the best solution to create the software. Software engineers need to analyze what is needed in order for this to be executed. The third stage includes the implementation phase. This is where the product is programmed by a team of developers. Coding tasks are broken down into smaller bits so it can be manageable to accomplish. The requirements for the coding tasks that are indentified is analyzed daily. The fourth stage includes the deployment phase. This is when software is developed by teams and tests are done on the different copies of software than the one that the users have access to. Production is the software that customers use. Other copies are to be in the build enviroment or the testing enviroment. Build and prod uction envirpments are important as they make sure the customer can use the software even if it is altered by changing or upgrading within the enviroments. This phase includes many tasks that move the lastest build copy to the production enviroment. Examples include packaging, installation and environment configuration. The last phase includes the maintenance phase. This includes the team fixing bugs, helping customers with issues, and managing software changes. Overall, system performance is monitored and the security performance of the the user experience. This is all to find out ways to to improve the software. 4 Main Attributes of Software Engineering

  • Efficiency: How efficientley a software product is utlizied within sysytem resources in order to perform at its full functionality.
  • Reliability: The software is ensured that it is consistently produced with efficient results with conditions that are consistent.
  • Reuseability: The capacity in which a software module is refered to, so it can be used across many different kinds of applications and systems.
  • Maintainability: When the software ability is altered to the point where it needs to be fixed or upgraded easily with requirements changing.
